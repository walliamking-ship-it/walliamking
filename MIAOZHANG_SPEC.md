## 七、权限系统

### 角色列表
| 角色 | 描述 |
|------|------|
| 老板 | 系统的最高权限者 |
| 超级业务员 | 可查看所有客户和销售数据，但只能编辑删除自己客户和销售数据 |
| 销售主管 | 可查看、编辑客户和销售数据信息 |
| 业务员 | 只可查看、编辑、删除自己的客户和销售数据 |
| 超级采购人员 | 可查看、编辑供应商和采购数据信息 |
| 采购主管 | 可查看、编辑供应商和采购数据信息 |
| 采购人员 | 只可查看、编辑供应商和采购数据信息 |
| 仓管人员 | 只可查看和操作产品的送货和收货信息 |
| 财务人员 | 能进行财务与业务活动 |
| 投资人员 | 可查看账户所有功能以及数据，不可以新增、编辑和删除 |

### 权限分类（细粒度）
每个模块（客户/销售单/采购单/产品/库存等）都有：
- 新建
- 查看自己新建
- 查看所有
- 批量导出
- 编辑自己新建
- 编辑所有
- 删除自己新建
- 删除所有

### 特殊权限
- 查看毛利
- 编辑产品进价/售价
- 整单折扣
- 优惠金额
- 设置赠品
- 查看客户/供应商电话/地址
- 云仓操作
- 自定义字段编辑
- 登录时间限制

## 十三、数据关联关系

```
客户(Customer)
  └── 销售单(SalesOrder) → 客户名称
  └── 收款单 → 客户名称
  └── 产品(Product) → 货号前缀(C01/C05等)

供应商(Vendor)
  └── 采购单(PurchaseOrder) → 供应商名称
  └── 付款单 → 供应商名称
  └── 加工单(ProcessingOrder) → 加工公司

产品(Product)
  └── 销售单明细 → 产品名称
  └── 采购单明细 → 产品名称
  └── 库存(Inventory) → 产品名称+规格
  └── 组成子产品(组合产品)

库存(Inventory)
  └── 仓库(仓库名称)
  └── 产品(通过产品名称关联)
  └── 批次/缸号
```
